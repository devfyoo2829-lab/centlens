# CentLens 와이어프레임 명세

이 폴더의 HTML 파일들은 Streamlit UI 구현을 위한 시각 명세입니다. Vercel 다크 테마를 기반으로 작성됐고, 마케터·크리에이티브 디렉터를 사용자로 가정한 한국어 인터페이스입니다.

## 파일 구성

- `page_1_upload.html` — 페이지 1: 새 영상 분석 (업로드 + 진행 상황)
- `page_2_detail.html` — 페이지 2: 영상 상세 (6축 점수 + A/B 관점 차이)
- `page_3_archive.html` — 페이지 3: 분석한 영상 모음 (카드 그리드 + 의미 검색)
- `README.md` — 본 문서 (디자인 토큰 + 툴팁 명세 + 변환 가이드)

브라우저에서 HTML 파일을 직접 열어 ⓘ 아이콘 hover 동작 확인 가능합니다.

## 페이지 구조

### 페이지 1: 새 영상 분석 (`/`)

```
┌─ 헤더 (CentLens 로고 + 부제 + 탭 네비) ─┐
├─ 페이지 타이틀 + 설명 ──────────────────┤
├─ [영상 입력] [분류·메타데이터] (2 컬럼) ─┤
├─ "분석 시작" 버튼 (전체 폭) ────────────┤
└─ 분석 진행 상황 ─────────────────────┤
   ├─ 영상 준비
   ├─ 6축 평가 (3×2 그리드, 병렬 강조)
   ├─ 교차 검증
   ├─ 등급 산출
   └─ 검색 인덱싱
```

### 페이지 2: 영상 상세 (`/?slug=...`)

```
┌─ 헤더 ──────────────────────────────────┐
├─ "← 분석한 영상 모음" 뒤로가기 ─────────┤
├─ 게임명 + 메타 + 종합 등급 (헤더 영역) ─┤
├─ 영상 플레이어 + 추출 5장면 (가로 배치)─┤
├─ 6축 평가 헤더 (가장 약한 축 표시) ───┤
├─ 변화 3축 카드 (3개 게이지) ───────────┤
├─ 연출 3축 카드 (3개 게이지) ───────────┤
├─ ⚠️ 관점 차이 분석 카드 (앰버 톤) ─────┤
│   └─ 1차 평가 vs 교차 검증 (2 컬럼)
├─ 분석 단계별 시간 (8개 셀 + 총합) ─────┤
└─ 마케터 코멘트 입력 ────────────────────┘
```

### 페이지 3: 분석한 영상 모음 (`/archive`)

```
┌─ 헤더 ──────────────────────────────────┐
├─ 페이지 타이틀 + 영상 수 ───────────────┤
├─ 통계 카드 4개 (전체/평균/최강/최약) ──┤
├─ 의미로 영상 찾기 ──────────────────────┤
│   ├─ 입력 + 찾기 버튼
│   └─ 예시 칩 (3개)
├─ 필터 (분류·등급·장르) + 정렬 ─────────┤
└─ 카드 그리드 (2 컬럼, 5편) ─────────────┘
   각 카드:
   ├─ 분류 뱃지 + 게임명 + 등급 뱃지
   ├─ 썸네일 (영상 thumbnail)
   ├─ 총점
   ├─ 6축 미니 막대 차트 (변화 3 + 연출 3)
   └─ 가장 약한 축 + 장르·퍼블리셔
```

## HTML → Streamlit 변환 가이드

Streamlit은 HTML/CSS 자유도가 제한적입니다. 다음 우선순위로 구현하세요.

### 정확히 재현해야 할 영역 (HTML/CSS 직접 삽입)

다음 영역은 시각적 인상이 핵심이라 `st.markdown(unsafe_allow_html=True)` 로 직접 HTML 작성:

1. **헤더 영역** — 로고 + 탭 네비게이션
2. **6축 게이지 카드** — 변화 3축, 연출 3축의 게이지와 라벨
3. **관점 차이 분석 카드** — A/B 비교 그리드 (페이지 2 핵심)
4. **6축 미니 차트** — 카드 안의 6개 막대 (페이지 3 카드)
5. **모든 ⓘ 툴팁** — Streamlit 기본 위젯에 hover 툴팁 못 붙임

이 영역들은 페이지마다 한 번만 CSS 정의하고 재사용.

### Streamlit 컴포넌트로 대체 가능 (간단)

다음은 Streamlit 위젯으로 만들어도 시각적 손실 적음:

1. **파일 업로드** — `st.file_uploader`
2. **YouTube URL 입력** — `st.text_input`
3. **분류 선택** — `st.radio` 또는 `st.segmented_control`
4. **게임명·장르 입력** — `st.text_input`
5. **"분석 시작" 버튼** — `st.button` (full width)
6. **마케터 코멘트** — `st.text_area`
7. **저장 버튼** — `st.button`

다만 기본 Streamlit 위젯은 시각이 시안과 다를 수 있어요. 다음 두 가지 중 선택:
- **A**: 위젯 그대로 사용 (빠름, 시안과 약간 차이)
- **B**: `st.markdown` + 폼 데이터를 URL 쿼리/세션 state로 직접 관리 (느림, 시안 정확 재현)

→ **A 추천** (시간 효율).

### Streamlit 한계로 타협 영역

다음은 시안의 시각적 디테일을 100% 재현 못 해요. 비슷하게 구현 후 진행:

1. **드래그 앤 드롭 영역** — Streamlit `st.file_uploader` 의 기본 룩 (버튼이지만 드래그도 됨). 시안의 점선 박스와 다름.
2. **호버 효과** — 카드, 버튼의 호버 색상 변화. CSS로 부분 재현 가능하지만 한계 있음.
3. **세그먼트 컨트롤 (분류 선택)** — Streamlit 1.29 이후 `st.segmented_control` 가 시안과 가장 가까움.
4. **6축 게이지 막대** — `st.progress` 는 시안과 다름. **plotly bar 차트**로 구현 권장.

### 라이브러리

다음 추가 라이브러리 사용:

```
streamlit-extras  # tooltip, badge 등 보조 컴포넌트
plotly            # 6축 게이지를 정밀 시각화
```

## 페이지 라우팅

Streamlit 멀티페이지 구조:

```
centlens/
├── app.py                              # 메인 진입점
└── pages/
    ├── 1_새_영상_분석.py
    ├── 2_영상_상세.py
    └── 3_분석한_영상_모음.py
```

페이지 2는 URL 쿼리 파라미터 `?slug=...` 로 영상 식별:

```python
slug = st.query_params.get("slug")
if not slug:
    st.error("영상을 선택해주세요.")
    st.stop()
record = repo.get_video(slug)
```

페이지 3 카드 클릭 시 페이지 2로 이동:

```python
if st.button("자세히 보기", key=f"detail_{record.slug}"):
    st.query_params["slug"] = record.slug
    st.switch_page("pages/2_영상_상세.py")
```
